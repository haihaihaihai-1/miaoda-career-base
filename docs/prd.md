# 需求文档

## 1. 应用概述

### 1.1 应用名称
职业路径导航器

### 1.2 应用描述
面向中国高职院校学生的职业发展规划工具。学生通过填写个人信息、技能评估和职业目标后,系统自动分析其与目标岗位的能力差距,并生成包含本校课程、GitHub实训项目、1+X证书考取建议的分阶段学习路径。应用采用深蓝色主题(#1a56db),整体风格简洁专业。

## 2. 用户与使用场景

### 2.1 目标用户
- 中国高职院校在校学生(大一至大三)
- 专业方向:软件技术、人工智能、工业机器人等技术类专业
- 有明确职业发展目标或希望获得职业规划指导的学生

### 2.2 核心使用场景
- 学生初次使用时填写个人信息和职业目标,获取定制化学习路径
- 学生查看当前学习进度和下一步行动建议
- 学生浏览热门岗位信息,了解行业要求
- 学生管理个人学习档案和证书记录

## 3. 页面结构与功能说明

### 3.1 页面结构

```
应用根页面
├── Tab 1 - 首页(职业路径)
├── Tab 2 - 引导页
├── Tab 3 - 发现页
└── Tab 4 - 我的
```

底部TabBar固定显示,包含4个标签页,用户可随时切换。

### 3.2 Tab 1 - 首页(职业路径)

#### 3.2.1 页面功能
展示学生当前的职业学习路径总览和学习进度。

#### 3.2.2 内容结构

**1+X证书报名提醒Banner**
- 当某个证书进度首次达到30%时,在页面顶部显示蓝色背景通知条
- 文案:「1+X {证书名}已达到可报名状态,建议关注近期考试安排」
- 5秒后自动消失,带淡出动画
- 使用localStorage记录已通知的证书ID,避免重复提醒

**顶部信息卡片**
- 显示学生姓名、专业、目标岗位
- 显示当前技能等级→目标技能等级(如:初级工L2→高级工L4)
- 显示路径匹配度百分比
- 右上角增加「分享路径」按钮

**学习里程碑时间轴**
- 以时间轴样式展示学习里程碑列表
- 每个里程碑显示:阶段名称、预估时长、完成状态
- 状态标识:
  - 已完成:绿色实心圆点
  - 进行中:蓝色圆点带光晕动画
  - 待开始:灰色空心圆点
- 点击里程碑打开底部抽屉(移动端)或右侧面板(桌面端)

**里程碑详情抽屉/面板**
- 显示该阶段的技能列表
- 显示学习资源(课程、GitHub项目、证书)
- 显示下一步建议
- 显示统计信息:总里程碑数、已完成数、进行中数、剩余数

**空状态**
- 当学生未生成路径时,显示「开始引导」按钮
- 点击按钮跳转至Tab 2引导页

#### 3.2.3 学习路径分享卡片

点击「分享路径」按钮后:
- 展示全屏遮罩层,中央显示分享预览卡片
- 卡片设计(白色圆角卡片,阴影):
  - 顶部:应用logo + 「职业路径导航器」文字
  - 中部:学生姓名 + 目标岗位 + 当前等级→目标等级
  - 关键数据:路径匹配度、总学时、预计周数
  - 3阶段摘要(每个阶段一行:阶段名·课程数·学时)
  - 底部:「扫码查看完整路径」 + 二维码占位图
- 「保存图片」按钮:截图为PNG并下载
- 「复制链接」按钮:复制应用URL到剪贴板
- 「关闭」按钮

### 3.3 Tab 2 - 引导页

#### 3.3.1 页面功能
通过6步问卷表单收集学生信息,用于生成职业学习路径。

#### 3.3.2 表单步骤

**步骤1:个人信息**
- 姓名输入框
- 学历选择(单选):中职/高职/本科
- 专业输入框

**步骤2:技能评估**
- 已掌握基础技能评分(1-5星评分)
- 每个技能评分实时显示标签:
  - 1-2星:橙色标签「需提升」
  - 3星:黄色标签「基础水平」
  - 4-5星:绿色标签「已掌握」
  - 标签在评分变化时实时更新,带动画过渡
- 已有技能多选(Java/Python/SQL/HTML/CSS/JavaScript等)
- 背景经历多选(实习经历/项目经验/竞赛获奖/无)

**步骤3:职业目标**
- 兴趣方向多选(Web开发/AI/数据分析/移动开发/运维/测试等)
- 学习目标输入框
- 目标岗位输入框

**步骤4:学习偏好**
- 学习方式偏好多选(视频/阅读/项目/测验)
- 学习风格单选(视觉型/听觉型/读写型/动手型)
- 学习节奏单选(快速/稳健/灵活)

**步骤5:时间投入**
- 每天学习时间选择(1小时以下/1-2小时/2-4小时/4小时以上)
- 学习周期选择(3个月/6个月/1年/2年)
- 每月预算选择(免费/500元以下/500-2000元/2000元以上)

**步骤6:确认提交**
- 显示填写信息总结
- 提交按钮

#### 3.3.3 交互功能
- 顶部显示进度条,标识当前步骤(1/6至6/6)
- 每步显示图标和名称
- 支持「上一步」「下一步」按钮切换
- 提供「一键填充示例数据」按钮,点击后自动填充演示学生数据(张三-软件技术-Java开发工程师的完整信息)
- 提交后跳转至Tab 1首页,显示生成的学习路径

### 3.4 Tab 3 - 发现页

#### 3.4.1 页面功能
展示热门职业岗位信息,帮助学生了解行业要求。

#### 3.4.2 内容结构

**搜索框**
- 支持按岗位名称搜索

**热门岗位卡片列表**
- 每个卡片显示:
  - 岗位名称(如:Java开发工程师/AI应用开发工程师/工业机器人运维工程师)
  - 要求技能等级(如:高级工L4)
  - 推荐专业(如:软件技术/人工智能技术应用)
  - 平均薪资范围
- 点击卡片弹出岗位详情弹窗

#### 3.4.3 岗位详情弹窗

点击岗位卡片后:
- 弹出详情弹窗
- 显示内容:
  - 岗位名称+薪资范围(大号字体)
  - 要求技能等级标签
  - 推荐专业
  - 核心技能要求列表(6-8个技能标签)
- 底部「查看学习路径」按钮:点击后自动在引导页填充该岗位为target_role并提示用户完成引导

### 3.5 Tab 4 - 我的

#### 3.5.1 页面功能
展示学生个人信息和学习档案。

#### 3.5.2 内容结构

**Dify AI配置入口卡片**
- 位于页面顶部
- 显示当前连接状态指示灯(绿色已连接/灰色未配置)
- 点击展开配置面板(内嵌表单,非跳转页面)
- 表单字段:
  - API地址输入框(placeholder:http://your-server:5001/v1)
  - API Key密码输入框
- 「测试连接」按钮:点击后向{VITE_DIFY_API_URL}/parameters发送GET请求验证连通性
  - 成功:绿灯+toast「连接成功」
  - 失败:红灯+toast「连接失败:{错误信息}」
- 配置保存到localStorage的dify_config键
- 初始值从.env的VITE_DIFY_API_URL和VITE_DIFY_API_KEY读取

**个人信息区**
- 头像
- 姓名
- 专业

**学习档案**
- 已完成课程列表(显示课程名称、学分、完成时间)
- 已获证书列表(显示证书名称、获取时间)

**学习统计**
- 总学时
- 完成率百分比

**设置按钮**
- 点击进入设置页面

## 4. 业务规则与逻辑

### 4.1 数据存储结构

应用需要存储以下数据:

**students表**
- id、name、major(专业)、grade(年级:大一/大二/大三)
- certificates(已获证书,JSON数组)
- skills(已掌握技能,JSON数组)
- career_interest(职业兴趣)
- target_role(目标岗位)
- current_level(当前职业技能等级1-8)
- weekly_hours(每周学习时间)
- created_at

**skill_levels表**(中国8级职业技能等级标准)
- id、level_num(1-8)
- level_name(学徒工/初级工/中级工/高级工/技师/高级技师/特级技师/首席技师)
- description、typical_duration、education_equivalent
- 预置8条技能等级记录

**courses表**(本校课程)
- id、major(所属专业)、name、semester、credits、hours
- skill_level(对应技能等级2-4)
- skills(培养的技能,JSON数组)
- 预置软件技术专业10门课程:Java程序设计、MySQL数据库、HTML5+CSS3、JavaScript、Spring Boot、Vue.js、软件测试、微服务架构、Git协作开发、项目实战
- 预置3个专业的课程数据

**career_paths表**
- id、student_id、target_role、total_weeks、total_hours
- stages(JSON:3个阶段,每阶段含课程列表、GitHub项目推荐、考证建议、预估学时)
- gap_items(JSON:能力差距清单)
- created_at

**milestones表**
- id、path_id、stage、title、description、duration
- status(completed/current/upcoming)
- progress(0-100)
- skills(JSON)
- resources(JSON)

### 4.2 能力差距分析逻辑

**输入**
- 学生已掌握技能列表
- 目标岗位

**处理流程**
1. 根据目标岗位从courses表匹配所需技能
2. 对比学生已掌握技能,计算缺失技能清单
3. 为每个缺失技能标注:
   - 技能名称
   - 类别
   - 当前状态(未掌握/部分掌握)
   - 所需等级(对应skill_levels表的level_num)
   - 优先级(高/中/低)
   - 预估学时

**输出**
- 缺失技能清单(存入career_paths表的gap_items字段)
- 雷达图数据(已掌握技能 vs 缺失技能的对比)

### 4.3 学习路径生成逻辑

**输入**
- 能力差距分析结果
- 学生当前等级
- 学生每周学习时间

**生成规则**

路径分为3个阶段:

**基础期(1-8周,目标:中级工L3)**
- 优先推荐低年级本校课程
- 推荐1+X初级证书
- 预估学时:80-120小时

**强化期(9-20周,目标:高级工L4)**
- 推荐核心专业课程
- 推荐GitHub项目实训:
  - spring-petclinic(7.5K★)
  - mall(75K★)
  - RuoYi-Vue(21K★)
- 推荐1+X中级证书
- 预估学时:150-200小时

**冲刺期(21-30周,目标:岗位胜任)**
- 推荐高阶课程
- 推荐项目实战
- 推荐职业资格认证
- 预估学时:100-150小时

**输出**
- 完整的3阶段学习路径(存入career_paths表的stages字段)
- 每个阶段包含:
  - 课程列表(从courses表匹配)
  - GitHub项目推荐
  - 考证建议
  - 预估学时
  - 里程碑检查点

### 4.4 1+X证书报名提醒逻辑

**触发条件**
- 学生引导提交后,首页已显示学习路径

**处理流程**
1. 从career_paths表的stages字段提取所有certifications
2. 对每个证书计算当前进度比例
3. 当某个证书进度首次达到30%时:
   - 在首页顶部显示蓝色背景Banner通知条
   - 文案:「1+X {证书名}已达到可报名状态,建议关注近期考试安排」
   - 5秒后自动消失,带淡出动画
4. 使用localStorage记录已通知的证书ID,避免重复提醒

### 4.5 Dify AI配置逻辑

**初始化**
- 从.env读取VITE_DIFY_API_URL和VITE_DIFY_API_KEY作为初始值
- 从localStorage的dify_config键读取用户配置

**测试连接**
- 向{VITE_DIFY_API_URL}/parameters发送GET请求
- 成功:状态指示灯变绿,显示toast「连接成功」
- 失败:状态指示灯变红,显示toast「连接失败:{错误信息}」

**配置保存**
- 将API地址和API Key保存到localStorage的dify_config键

- 专业:软件技术
- 年级:大二
- 已掌握技能:Java、HTML/CSS、SQL
- 目标岗位:Java开发工程师
- 当前等级:初级工(L2)

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 学生未填写引导问卷 | 首页显示空状态,引导用户点击「开始引导」按钮 |
| 学生填写问卷时中途退出 | 保存当前步骤的填写内容,下次进入时恢复 |
| 学生已掌握技能覆盖目标岗位所有要求 | 显示「您已具备该岗位所需技能」,建议选择更高目标 |
| 目标岗位在courses表中无匹配课程 | 提示「暂无该岗位的课程数据,请选择其他岗位」 |
| 学生修改个人信息或目标岗位 | 重新生成学习路径,原路径进度清零 |
| 搜索岗位无结果 | 显示「未找到相关岗位」 |
| Dify AI配置测试连接失败 | 状态指示灯变红,显示toast「连接失败:{错误信息}」 |
| 分享路径时截图失败 | 显示toast「截图失败,请重试」 |
| 复制链接到剪贴板失败 | 显示toast「复制失败,请手动复制」 |

## 7. 验收标准

1. 学生打开应用,首页显示空状态和「开始引导」按钮
2. 学生点击「开始引导」,进入引导页填写6步问卷(或点击「一键填充示例数据」)
3. 学生在步骤2技能评估中对某技能评分,实时显示对应标签(需提升/基础水平/已掌握)
4. 学生提交问卷后,系统生成3阶段学习路径并跳转至首页
5. 首页顶部卡片显示学生信息、目标岗位、当前等级→目标等级、路径匹配度
6. 首页时间轴显示3个学习里程碑,状态为「进行中」「待开始」「待开始」
7. 学生点击第一个里程碑,打开详情抽屉/面板,查看该阶段的技能列表、学习资源和下一步建议
8. 学生点击「分享路径」按钮,查看分享预览卡片,点击「保存图片」下载PNG图片
9. 学生切换至Tab 3发现页,点击某个岗位卡片,弹出岗位详情弹窗,查看核心技能要求
10. 学生切换至Tab 4我的页面,点击「Dify AI配置」入口,填写API地址和API Key,点击「测试连接」,显示连接成功提示
